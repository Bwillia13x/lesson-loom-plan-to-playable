# Build Execution Brief — Lesson Loom v2

_Last updated: 2026-05-27_  
_Primary agent: Cursor Composer 2.5_

## Purpose

This is the short, high-control briefing to paste into the agent before build work begins. It compresses the full context pack into the exact execution target.

## Build the thing, not the platform

Build a polished one-page interactive prototype for **Lesson Loom — Plan to Playable**.

The prototype must show this transformation clearly:

```text
trusted lesson plan → teaching signal → interactive student app → teacher console → review/export pack
```

The build should feel like a finished product vision for a contest submission, not a backend product, LMS, or real curriculum system.

## Primary demo

Use exactly one lesson:

**Fraction Garden — Grade 5 equivalent fractions**

A teacher starts with a plain lesson plan. Lesson Loom turns it into a visual student activity where learners compare equivalent fractions using garden beds and fraction tiles.

## Non-negotiable scope

Do:

- build the single-page prototype;
- use deterministic static data;
- create beautiful UI and interactions;
- include Teacher and Student modes;
- include safety/review language;
- include a Stitch/process/export section;
- make it responsive and accessible enough for a public demo.

Do not:

- add backend/auth/database;
- add real AI calls;
- add real student accounts or student data;
- add grading automation;
- add LMS integration;
- create a generic lesson-plan generator;
- claim official curriculum alignment, compliance, or district readiness.

## Build priority order

1. **Hero clarity** — user understands in 10 seconds.
2. **Student App Preview** — visual centerpiece and screenshot hero.
3. **Lesson Weave interaction** — memorable transformation mechanic.
4. **Teacher Console** — proves practical classroom utility.
5. **Review & Export Pack** — proves responsible, Stitch-native workflow.
6. **Responsive polish** — no mobile breakage.

## Minimum viable interaction set

- “Weave lesson” activates the transformation.
- Teaching Signal cards reveal or become active after weaving.
- Student/Teacher mode toggle switches the workspace.
- Fraction tiles can be selected.
- Selecting three equivalent half-value tiles triggers success feedback.
- Support/Core/Extend lanes switch content.
- Export cards show copy feedback.
- Reduced-motion setting disables decorative animation.

## Design target

Warm, premium classroom studio.

Use:

- ivory paper background;
- deep navy and graphite type;
- sage/garden green;
- restrained lavender-blue;
- pale gold success accents;
- thin linework;
- tactile fraction tiles;
- woven path motif.

Avoid:

- childish primary colors;
- neon/cyberpunk;
- robots/AI brains;
- stock teacher/student imagery;
- generic SaaS dashboard styling.

## Definition of done

The build is done when:

- the concept is clear from the hero screenshot alone;
- the Fraction Garden student app feels interactive and visually rich;
- Teacher Console and UDL/differentiation make the product credible;
- Review & Safety avoids risky K–12 claims;
- Export Pack makes the Stitch workflow visible;
- desktop and mobile layouts work;
- build command passes;
- remaining limitations are listed honestly.

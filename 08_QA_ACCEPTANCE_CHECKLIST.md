# QA and Acceptance Checklist — Lesson Loom

## Product acceptance

- [ ] The concept is clear within 30 seconds.
- [ ] The hero explains “lesson plan → interactive classroom app.”
- [ ] The demo focuses on one lesson: Fraction Garden.
- [ ] The prototype does not feel like a generic SaaS landing page.
- [ ] The output feels useful to a teacher.
- [ ] The student app feels interactive and visually rich.
- [ ] The teacher console feels practical.
- [ ] The export pack ties the project back to Stitch and build handoff.

## Functional checks

- [ ] **Weave lesson** button changes app state.
- [ ] Teaching signal cards reveal or activate after weave.
- [ ] Lesson Weave path/stages activate in order or appear active.
- [ ] Student/Teacher toggle works.
- [ ] Fraction tile selection works.
- [ ] Equivalent fraction success state works.
- [ ] Hint button works or reveals hint text.
- [ ] Support/Core/Extend toggle works.
- [ ] Export Pack copy buttons show feedback.
- [ ] Anchor links / section CTAs scroll correctly if implemented.

## Visual checks

- [ ] Warm ivory background is applied consistently.
- [ ] Typography hierarchy is clear.
- [ ] Cards have refined spacing, borders, and shadows.
- [ ] Fraction Garden is the visual centerpiece.
- [ ] Motion is restrained and premium.
- [ ] No cluttered dashboard feel.
- [ ] No childish primary-color overload.
- [ ] No generic robot/AI-brain imagery.
- [ ] Hero is screenshot-worthy.

## Responsive checks

Test widths:

- [ ] 1440px desktop.
- [ ] 1280px laptop.
- [ ] 1024px tablet landscape.
- [ ] 768px tablet portrait.
- [ ] 430px mobile.
- [ ] 390px mobile.

For each:

- [ ] No horizontal scroll.
- [ ] Text remains readable.
- [ ] Fraction tiles wrap correctly.
- [ ] Mode toggle remains usable.
- [ ] Export cards stack gracefully.
- [ ] Hero visual does not crush copy.

## Accessibility checks

- [ ] Semantic heading order.
- [ ] Real buttons for interactions.
- [ ] Visible keyboard focus.
- [ ] Controls can be tabbed to.
- [ ] Interactive fraction tiles have accessible labels.
- [ ] Color is not the only indicator of selection/success.
- [ ] Contrast is acceptable.
- [ ] Reduced-motion preference is respected.
- [ ] Text alternatives or labels exist for visual-only concepts.

## Claim-safety checks

- [ ] Does not claim official curriculum generation.
- [ ] Does not claim teacher replacement.
- [ ] Does not claim district approval.
- [ ] Does not claim full privacy compliance.
- [ ] Does not imply student profiling.
- [ ] Does not automate grading.
- [ ] Review & Safety section is visible.
- [ ] “Teacher review required” language is present.
- [ ] “No student data required” language is present.
- [ ] “Printable fallback” language is present.

## Performance checks

- [ ] App builds successfully.
- [ ] No console errors in normal flow.
- [ ] No huge unnecessary assets.
- [ ] No blocking external scripts unless necessary.
- [ ] CSS animations do not cause obvious jank.

## Browser checks

At minimum:

- [ ] Chrome.
- [ ] Safari if available.
- [ ] Mobile Safari if available.

## Submission checks

- [ ] Live URL works.
- [ ] Page title is set.
- [ ] Meta description is set.
- [ ] Screenshots captured.
- [ ] 60–90 second walkthrough recorded.
- [ ] Contra submission copy ready.
- [ ] Source/credits clear.
- [ ] Final challenge rules checked manually before posting.

## Definition of done

The prototype is done when it can be shown cold to a judge and they can say:

> “I get it. A teacher’s static lesson became a beautiful interactive classroom experience, and the teacher still controls review and use.”

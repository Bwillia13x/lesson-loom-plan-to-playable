# References and Source Notes

This file captures the source assumptions behind the Lesson Loom context pack.

## Official/current sources to verify before final submission

### Google Stitch

- Stitch official site: https://stitch.withgoogle.com/
- Stitch prompting docs: https://stitch.withgoogle.com/docs/learn/prompting/
- Google blog: “Introducing vibe design with Stitch” — https://blog.google/innovation-and-ai/models-and-research/google-labs/stitch-ai-ui-design/
- Google blog: DESIGN.md in Stitch — https://blog.google/innovation-and-ai/models-and-research/google-labs/stitch-design-md/
- Google AI Developers Forum prompt guide: https://discuss.ai.google.dev/t/stitch-prompt-guide/83844

Working assumption:

- Stitch is useful for high-fidelity UI exploration from natural language.
- The project should visibly show prompt-to-interface transformation and design-system continuity.

### Contra Stitch Challenge

- Challenge hub: https://contra.com/community/topic/stitchchallenge
- Guidelines route: https://contra.com/community/topic/stitchchallenge/guidelines

Working assumption:

- Final rules, deadline, allowed tools, and submission format must be manually checked while logged in before posting.

### Cursor / Composer

- Cursor homepage: https://cursor.com/
- Composer 2 announcement: https://cursor.com/blog/composer-2
- Composer 2 technical report: https://cursor.com/blog/composer-2-technical-report

Working assumption:

- Composer 2/2.5 is suitable as the primary Cursor implementation model for multi-file agentic coding work.
- The agent should work in scoped implementation passes, not one giant uncontrolled rewrite.

### UDL / learning design

- CAST UDL Guidelines: https://udlguidelines.cast.org/

Working assumption:

- Lesson Loom can use UDL-informed language around multiple means of engagement, representation, and action/expression.
- Do not claim formal certification.

### Canadian education privacy

- Office of the Privacy Commissioner of Canada / FPT Privacy Regulators: Protecting the privacy of children and youth through responsible use of educational technologies in the classroom: https://www.priv.gc.ca/en/about-the-opc/what-we-do/provincial-and-territorial-collaboration/joint-resolutions-with-provinces-and-territories/res_20251008_edtech/

Working assumption:

- Children’s information should be treated as sensitive.
- The contest prototype should use no student personal data.
- Privacy-protective defaults and data minimization should be visible in the UI.

### Alberta curriculum

- Alberta K–12 curriculum and programs of study: https://www.alberta.ca/k-12-curriculum-programs-of-study
- LearnAlberta curriculum: https://curriculum.learnalberta.ca/curriculum/en

Working assumption:

- Lesson Loom should not claim official curriculum generation.
- If curriculum alignment is shown, it should be framed as teacher-verifiable draft alignment.

## PrairieSignal project-source assumptions

This build should be treated as a PrairieSignal Labs prototype that supports Research and Edge.

Key internal assumptions:

- PrairieSignal uses four arms: Studio, Edge, Research, Labs.
- Labs prototypes future options but should not distract from revenue without a clear reason.
- Research and Labs should make Studio and Edge easier to sell.
- Proof assets must be labeled accurately.
- Do not overclaim prototype readiness.
- High autonomy inside bounded workflows; human approval for external actions.

## Claim language summary

Use:

- prototype;
- classroom-ready draft;
- teacher-reviewed;
- curriculum-aligned draft;
- no student data required;
- printable fallback;
- UDL-informed.

Avoid:

- official curriculum generator;
- fully compliant;
- district-ready;
- replaces teachers;
- automated grading;
- proven learning outcomes.
